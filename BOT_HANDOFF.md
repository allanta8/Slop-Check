# Slop Check Twitter Bot — Build Handoff

Context for the next Claude Code session. The bot **lives in the `viewft` repo**, not here. This `slop-check` repo only owns the skill (`SKILL.md`) the bot calls into.

## Goal

A Twitter bot that, when tagged in a reply or quote, runs the slop-check skill on the original post and replies with a punchy short tweet + link to a full report.

## Decisions locked in

| Question | Answer |
|---|---|
| Twitter API tier | Basic ($200/mo) — user provides creds |
| Mention discovery | Polling `GET /2/users/:id/mentions` every 5 min |
| Trigger types | Reply mentioning bot, quote tweet mentioning bot |
| Bare mention with no parent | Reply: "tag me on the post you want checked" |
| Image-only / video-only parent | Reply: "no text to check" |
| Stack | Node.js + TypeScript, `twitter-api-v2`, `@anthropic-ai/sdk` |
| Model | `claude-sonnet-4-6` |
| State | `state.json` with `since_id` (last processed mention) |
| Idempotency | `since_id` updated only after successful reply |
| Runtime | GitHub Actions cron `*/5 * * * *` (one-shot process per run) |
| Full report hosting | viewft.com (post via viewft API — contract TBD by user) |
| Self-mentions | Ignored |
| Abuse guard | Cap N mentions per poll; ignore bot's own author_id |

## Reply format (≤280 chars)

```
🚨 Slop: 5/10 — REWRITE

"We're early" with zero specifics on why this thing is early. Sounds opinionated, says nothing.

📊 voice 4 · humanness 3 · no_ai_tells 6
🔗 slopcheck.viewft.com/r/1893…
```

- **Humanizer line** — punchy 1-sentence roast in human voice, generated by Claude
- **Mini scorecard** — 3 lowest-scoring dimensions (most damning)
- **Link** — full report on viewft

## Claude call shape

System prompt = `SKILL.md` from this repo (vendor it into the bot repo or fetch raw from GitHub on cold start). Wrap user input with an instruction to return JSON instead of the markdown report.

Expected response:
```json
{
  "overall": 5,
  "verdict": "REWRITE",
  "humanizer": "...",
  "lowest_dims": [
    {"name": "humanness", "score": 3},
    {"name": "voice_match", "score": 4},
    {"name": "no_ai_tells", "score": 6}
  ],
  "full_report_html": "..."
}
```

One Claude call produces both the tweet bits and the full report body.

## Per-mention flow

1. `GET /2/users/:id/mentions?since_id=...&expansions=referenced_tweets.id&tweet.fields=text,author_id,referenced_tweets`
2. Resolve original post:
   - Reply → `referenced_tweets[type=replied_to]`
   - Quote → `referenced_tweets[type=quoted]`
   - Neither → reply with hint, skip slop check
3. Fetch parent tweet's full text + author handle
4. Call Claude with `SKILL.md` system prompt → JSON above
5. POST report HTML to viewft API → get back report URL
6. Build ≤280 char reply via `format.ts`
7. `POST /2/tweets` with `in_reply_to_tweet_id = mention.id`
8. Update `state.json` `since_id`

## Repo layout (in `viewft` repo, suggested path `apps/slop-bot/`)

```

  twitter.ts          # mentions fetch, parent resolution, reply post
  slop.ts             # Claude call, returns structured JSON
  publish.ts          # POST report HTML to viewft, return URL
  format.ts           # builds ≤280 char reply

```

## Out of scope for v1

- Image / video OCR on parent tweets
- Multi-tweet threaded replies
- Web dashboard / admin UI
- Bot scoring its own replies
- Rate-limit backoff beyond Twitter SDK defaults
